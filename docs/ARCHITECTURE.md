# 架构说明

## 系统架构

```
PDF输入
  ↓
OCR识别（PaddleOCR）
  ↓
OCR JSON生成
  ↓
目录检测
  ↓
  ├─ 有目录 → 目录拆分（零成本）
  └─ 无目录 → LLM智能拆分
       ↓
    后处理
       ↓
    输出PDF文件
```

## 核心组件

### 1. OCR模块

**类**: `LLMDocumentSplitter._init_ocr()`

**功能**:
- 自动检测GPU可用性
- 初始化PaddleOCR引擎
- 支持GPU/CPU自动切换

**优化**:
- 图像缩放降低内存使用
- 批处理提高效率
- 简化的JSON输出（仅关键信息）

### 2. 目录检测模块

**类**: `LLMDocumentSplitter._detect_toc_page()`, `_extract_toc_entries()`

**功能**:
- 检测PDF中的目录页
- 提取目录条目和页码
- 支持多种目录格式

**优势**:
- 零成本（不调用LLM）
- 精确拆分
- 快速处理

### 3. LLM拆分模块

**类**: `LLMDocumentSplitter._build_system_prompt()`, `_build_user_prompt()`, `_call_llm()`

**功能**:
- 构建系统提示词（拆分规则）
- 构建用户提示词（页面信息）
- 调用LLM API获取拆分建议

**优化**:
- Token优化（仅关键信息）
- 紧凑的提示词格式
- 大文件自动截断

### 4. 后处理模块

**类**: `LLMDocumentSplitter._post_process_splits()`

**功能**:
- 修正重叠页面
- 合并补充材料
- 验证页面覆盖
- 规范化文件名

## 数据流

### OCR JSON格式

```json
{
  "pages": [
    {
      "page_num": 1,
      "page_height": 842.0,
      "texts": [
        {"text": "文本内容", "y": 100.5}
      ]
    }
  ]
}
```

**优化**: 仅包含关键信息（文本和Y坐标），大幅减少文件大小。

### 提示词格式

**系统提示词**:
- 拆分规则说明
- 文档类型特定规则
- 标题提取要求

**用户提示词**:
```
页码|页眉|前3行文本|页面类型
```

**优化**: 紧凑格式，仅包含关键信息。

### LLM响应格式

```json
{
  "splits": [
    {
      "start_page": 1,
      "end_page": 10,
      "title": "文档标题"
    }
  ]
}
```

## 配置系统

### 关键词配置

**类常量**:
- `SUPPLEMENT_KEYWORDS`: 补充材料关键词
- `PAGE_TYPE_KEYWORDS`: 页面类型关键词

**优势**:
- 易于扩展
- 不依赖硬编码
- 支持多语言

### 文档类型配置

**类型**:
- `general`: 通用文档
- `legal`: 法律文书
- `academic`: 学术论文

**影响**:
- 系统提示词内容
- 后处理策略
- 页面类型检测

## 性能优化

### 内存优化

1. **图像缩放**: 降低 `image_scale` 减少内存使用
2. **批处理大小**: 降低 `batch_size` 减少并发内存
3. **及时释放**: 使用 `gc.collect()` 释放内存

### Token优化

1. **简化OCR JSON**: 仅包含关键信息
2. **紧凑提示词**: 仅页眉和前3行文本
3. **自动截断**: 大文件自动截断，确保所有页面都被处理

### 速度优化

1. **GPU加速**: 自动使用GPU加速OCR
2. **目录优先**: 有目录时优先使用目录拆分
3. **批处理**: OCR批处理提高效率

## 扩展性

### 添加新的文档类型

1. 在 `_build_system_prompt()` 中添加新类型的规则
2. 在 `_detect_academic_page_type()` 中添加页面类型检测
3. 在 `_post_process_splits()` 中添加后处理逻辑

### 添加新的关键词

1. 在类常量中添加关键词
2. 在相关方法中使用配置的关键词

### 自定义拆分规则

1. 修改 `_build_system_prompt()` 中的规则说明
2. 修改 `_post_process_splits()` 中的后处理逻辑

## 错误处理

### OCR错误

- 自动降级到CPU
- 记录错误日志
- 继续处理其他页面

### LLM错误

- 重试机制
- 错误日志
- 降级处理（如果可能）

### 文件错误

- 验证文件存在
- 检查文件格式
- 错误提示

## 日志系统

使用Python标准 `logging` 模块：

- `INFO`: 正常流程信息
- `WARNING`: 警告信息（如GPU不可用）
- `ERROR`: 错误信息
- `DEBUG`: 调试信息（可选）
